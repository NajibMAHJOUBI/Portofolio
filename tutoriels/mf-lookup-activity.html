Objectifs du tutoriel

À la fin de ce tutoriel, vous serez capable de :

  • ✓ Comprendre le fonctionnement de l'activité Lookup
  • ✓ Configurer une activité Lookup pour lire des données de configuration
  • ✓ Utiliser les résultats de Lookup pour des activités itératives

Prérequis : Un capacité Microsoft Fabric (niveau gratuit suffit) et des notions de base du cloud computing.

Qu'est-ce qu'une activité Lookup ?

Un des points fort d'une plateforme de données est de permettre de créer des modules réutilisables. Elle doit intégrer l'intelligence opérationnelle, de la configuration dynamique et des décisions pilotées par la donnée. Pour cela, il est notamment nécessaire de pouvoir récupérer des paramètres sans avoir à les coder en durs. Et c'est précisément le rôle de l'activité Lookup dans Microsoft Fabric. L'activité Lookup dans Microsoft Fabric permet de récupérer des métadonnées depuis une source de données (SQL, Lakehouse, Warehouse, etc.) et de les réutiliser dans les pipelines de données tout en adaptant leur comportement à l'exécution. Utilisé efficacement, une activité Lookup permet de concevoir des pipelines dynamiques, configurables et alignés avec les meilleures pratiques en Data Engineering modernes. Il s'agit d'un composant d'orchestration puissant qui permet d'exposer son résultat sous forme de payload JSON utilisable par les autres activités du pipeline.

Scénario de l'exemple

Nous allons présenter un exemple simple d'illustration d'une activité LookUp dans un Pipeline de Données. Il s'agit de déplacer des fichiers entre deux dossiers du Lakehouse (ingest// et archive/). Apres avoir été déplacés, ces fichiers doivent être renommés selon une règle définie dans une table.

Dans les paragraphes suivants, nous allons décrire pas à pas les étapes de mise en place de ce scénario. L'objectif est de montrer au travers de cet exemple comment mettre en place une activité Lookup.

Création d'un Workspace

La première étape consiste à créer un Workspace dans Microsoft Fabric. Un Workspace est un environnement de travail collaboratif qui permet de regrouper des ressources, des données et des pipelines de données. Pour créer un Workspace, suivez les étapes suivantes :

  1. Cliquer sur Home depuis la volet gauche de l'interface de Microsoft Fabric
  2. Cliquer sur l'icône + New Workspace
  3. Dans le volet droit qui s'ouvre, entrer un nom dans le champ Name. Ici nous allons le nommer TutorielLookupActivity.
  4. Entrer une description dans le champ Description [Optionnel]
  5. Cliquer sur le bouton Apply pour lancer la création du Workspace

Création d'un Lakehouse

La deuxième étape de notre scénario consiste à créer un Lakehouse dans Microsoft Fabric. Un Lakehouse est une ressource de stockage de données qui permet de stocker des données structurées et non structurées. Pour créer un Lakehouse, suivez les étapes suivantes :

  1. Cliquer sur le bouton + New item
  2. Depuis le volet droit qui s'ouvre, taper le terme "Lakehouse" dans le champ de recherche
  3. Cliquer sur l'icône Lakehouse

Cela va ouvrir une fenêtre depuis laquelle nous allons pouvoir renseigner un nom pour ce Lakehouse en suivant les étapes suivantes :

  1. Renseigner un nom dans le champ Name (ici nous allons le nommer Lookup_Lk)
  2. Dans le champ Location, choisir le Workspace que nous avons créé
  3. Cliquer sur le bouton Create

Chargement de fichiers dans le Lakehouse

La troisième étape de notre scénario consiste à charger les fichiers utiles à ce tutoriel dans le Lakehouse que nous avons créé. Pour cela, nous allons créer deux dossiers dans ce Lakehouse : ingest/ et archive/. Nous allons ensuite charger des fichiers dans le dossier ingest/. Pour cela, suivez les étapes suivantes :

  1. Depuis l'interface du Lakehouse, cliquer sur le bouton + New et choisir l'option New folder
  2. Cliquer sur l'option **Upload** puis sur **Upload files**

Ces actions vont ouvrir un volet droit :

  1. Cliquer dans le champ pour ouvrir un explorer de fichier

Depuis l'explorer de fichier, réaliser les étapes suivantes :

  1. Sélectionner le fichier params.csv que vous pouvez trouver dans les ressources de ce tutoriel
  2. Cliquer sur le bouton **Open**

Après avoir cliqué sur le bouton Open, vous devriez voir le fichier sélectionné dans le volet droit de Microsoft Fabric.

  1. Constater que le fichier choisi est bien listé
  2. Cliquer sur le bouton **Upload** pour lancer le téléchargement du fichier

Bonnes pratiques S3

  • ✅ Nommage des buckets : Utilisez des noms uniques, en minuscules, sans underscores. Exemple: "mon-appli-data-2024"
  • ✅ Organisation : Structurez avec des préfixes (dossiers) comme "logs/2024/10/"
  • ✅ Sécurité : Activez le chiffrement par défaut et bloquez l'accès public sauf si nécessaire
  • ✅ Cycle de vie : Configurez des règles pour déplacer automatiquement vers Glacier après 30 jours
  • ✅ Monitoring : Activez les logs d'accès et Amazon CloudWatch
